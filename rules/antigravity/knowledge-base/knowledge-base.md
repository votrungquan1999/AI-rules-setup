# Knowledge Base

- **Retrieve first.** Before non-trivial work, search the knowledge base; if a canonical entry answers the problem, apply it and cite its id.
- **Capture what generalizes.** After solving a non-obvious problem, learning a surprising fact, or finding a reusable pattern — anything worth reusing later or by a teammate — capture it. Skip trivial / one-off things; they only add review noise.
- **Memory is for always-true project facts only.** Use it sparingly for conventions/constraints that must always be known here; keep each to 1–2 lines.
- **Drafts are not canonical.** Everything you capture is a draft pending human review — never assume it's live.
- **Commands live in the `knowledge-base` skill.** See it for the `kb` CLI mechanics: `search` / `get` / `capture` / `update` / `delete`, scoping (`--scope` / `--global`), and note-body passing.
