# Knowledge Base

- **Retrieve first.** Before non-trivial work, `kb_search` the knowledge base. If a
  canonical doc answers the problem, apply it and cite its id.
- **Capture when it's worth keeping.** After solving a non-obvious problem, finding a
  reusable pattern, or learning a surprising fact, draft a note with the matching tool:
  `capture_question` / `capture_til` / `capture_blueprint`. Do NOT capture trivial or
  one-off things — they only add review noise.
- **Memory is for always-true project facts only.** Use `capture_memory` sparingly for
  conventions/constraints that must always be known here. Keep each to 1–2 lines.
- **Drafts are not canonical.** Everything you capture is a draft pending human review;
  never assume it's live.
